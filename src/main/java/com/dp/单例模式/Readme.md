# 单例模式

## 静态常量实现

优势:

* 依赖jvm类加载机制,只做一次实例化.
* 由jvm保证线程安全.
* 实现简单

劣势:

* 类加载即实例化
* 可以通过反射获取类的实例,无法保证jvm中只有一个类的实例

## 静态语句块实现

与静态常量实现类似

## 内部类实现

内部类在使用时才会被加载到jvm中,避免了无效加载导致的对象实例化.

## synchronized实现

优势:

* 线程安全
* 懒汉模式,使用时才创建

劣势:

* 高并发获取实例时,性能差

## 双重检查

优势:

* 线程安全
* 并发度高

劣势:

* 仍旧可以通过反射获得多个实例对象

## 枚举实现

优势:

* 线程安全
* 即便通过反射获得实例,仍旧是同一个,即在同一个jvm中不会出现多个实例.

劣势:

* 语义上不符合习惯.(其实无所谓...^_^)
